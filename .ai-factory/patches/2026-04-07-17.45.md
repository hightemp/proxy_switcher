# Proxy lifecycle, auth validation, and lint blockers

**Date:** 2026-04-07 17:45
**Files:** `ProxyServer.kt`, `ProxyService.kt`, `HomeScreen.kt`, `AddEditProxyScreen.kt`, `ProxyFormValidator.kt`, `ProxyProtocolUtils.kt`, manifest/backup config, Gradle config, tests
**Severity:** high

## Problem

The app had lint-blocking Android API/permission issues and several runtime risks in proxy startup and credential handling. The service could apply the system proxy before the local server had actually bound to port 8080, the local server listened on all interfaces, active tunnels were not force-closed on stop, blank proxy passwords could break SOCKS5 auth, and HTTP proxy auth header injection converted request bytes through `String`.

## Root Cause

The proxy service assumed `ProxyServer.start()` was synchronous even though bind happened in an internal coroutine. Form validation trusted non-empty strings and silently defaulted bad ports to 8080. Header modification mixed HTTP headers and request body bytes as one text string. Android lint blockers were from direct API 26 foreground-service usage under `minSdk=24` and an intentional ADB-granted protected permission that was not documented for lint.

## Solution

Made `ProxyServer.start()` suspend until bind succeeds and return a Boolean result, bound the server to loopback, tracked sockets so `stop()` can close active tunnels, and had `ProxyService` apply system proxy only after successful bind. Added a service status broadcast for UI state. Added pure helpers for proxy form validation and HTTP auth header injection, using byte-safe header insertion and blank-password-safe auth. Disabled/excluded backup for proxy credentials and fixed Android lint blockers. Added focused JVM tests for form validation and header insertion.

## Prevention

- Treat service/server lifecycle APIs as result-bearing operations when later steps mutate system settings.
- Keep byte protocol helpers pure and testable; do not modify HTTP bodies through text conversion.
- Validate user-entered network endpoints explicitly; never silently substitute default ports.
- Bind local-only utilities to loopback unless remote access is an explicit feature.
- Run `lintDebug` alongside unit tests for Android service/manifest changes.
